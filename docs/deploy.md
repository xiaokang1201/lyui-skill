## 微项目部署及上线流程

目前自动化部署流程还在计划建设中...

所以部署均需要手动打包构建项目

### 测试环境部署

以新项目为例

运行命令执行一下打包

``` shell
yarn build
```

将打包生成的 dist目录 提交到 gitLab；

前端人员告知项目相关测试同事：要部署一下前端的项目，git地址、要部署哪个分支、项目名称；

测试同事需要在 Jenkins 中新建项目（前端项目的类型，测试同事一般都知道怎么建，如果不清楚，让测试同事询问上级）；

测试同事需要在对应的服务器中新建文件夹；

通过 Jenkins 部署成功！

> PS：特别注意！！！提醒测试同事，Jenkins 中新建的项目名称，需要与我们前端项目的项目名称 + gitLab项目名称 保持一致（不一致可能会导致部署正式环境失败）

通过 itest 环境访问项目链接

进行自测 （冒烟）

项目转测

### 正式环境部署

找测试同事获取 scp 的链接（如果测试同事不清楚，让询问上级了解）

将 scp 链接发送给运维同事，同理新项目，项目名也需要发送（运维同事需要根据项目名，在正式的服务器新建对应目录）

等待运维同事部署完毕...

联系相关人员配置正式环境菜单（一般是产品，把访问链接、新项目的菜单编码发给产品），正式环境的菜单编码需要与测试环境保持一致；

编码不一致的情况下，需要调整页面访问链接，总之就是需要满足 URL 匹配规则

```
/lyj-menu/一级菜单编码/二级菜单编号?submenu=三级菜单编号#/微项目路由地址
```

拿到正式环境链接之后，前往[正式环境的微项目配置页](http://i.leyoujia.com/lyj-menu/config)，点击新增子项目按钮，输入配置信息

正式环境访问项目链接，检查是否部署成功！

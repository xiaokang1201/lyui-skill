# LyUI Skill 架构设计文档

## 概述

LyUI Skill 是一个为 Cursor/AI 编程助手设计的组件库技能包，提供智能组件检索、文档索引和代码示例功能。

## 核心模块详解

### 1. SearchIndex (搜索索引)

**职责**: 提供高效的组件搜索能力

**核心算法**:
- **倒排索引 (Inverted Index)**: O(1) 关键词查找
- **n-gram 模糊匹配**: 支持拼写容错
- **拼音搜索**: 支持中文拼音输入
- **加权评分**: ID > 显示名 > 关键词 > 模糊匹配

**数据结构**:
```typescript
interface InvertedIndex {
  keywordToIds: Map<string, Set<string>>;  // 关键词 -> 组件ID集合
  idToKeywords: Map<string, Set<string>>;  // 组件ID -> 关键词集合
}
```

**搜索流程**:
1. 精确ID匹配 (快速路径)
2. 拼音匹配
3. n-gram 模糊匹配
4. 前缀匹配
5. 拼音前缀匹配
6. 评分排序

### 2. DocParser (文档解析)

**职责**: 解析 Markdown 文档，提取结构化信息

**解析内容**:
- 文档标题和描述
- 代码示例 (:::demo 语法)
- API 表格 (Attributes/Events/Slots)
- 章节结构

**输出格式**:
```typescript
interface ParsedDoc {
  id: string;
  title: string;
  description: string;
  sections: DocSection[];
  codeExamples: CodeExample[];
  totalLines: number;
}
```

### 3. Pinyin (拼音支持)

**职责**: 支持拼音搜索

**实现方式**:
- 预定义常用词汇拼音映射表
- 支持完整拼音和首字母匹配
- 集成到 SearchIndex 的索引构建和搜索流程

### 4. Cache (缓存)

**职责**: 缓存搜索结果，提升性能

**实现**: LRU (Least Recently Used) 缓存
- 最大容量: 50条
- 过期时间: 5分钟
- 淘汰策略: 最近最少使用

### 5. ErrorHandler (错误处理)

**职责**: 统一的错误类型和处理

**错误类型**:
- `LyUIError`: 基础错误
- `SearchError`: 搜索错误
- `DocParseError`: 文档解析错误
- `ComponentNotFoundError`: 组件未找到
- `ValidationError`: 参数验证错误

## 数据流

### 搜索流程

```
用户输入 -> 验证 -> 检查缓存 -> 构建候选集 -> 评分排序 -> 返回结果
                │                    │
                v                    v
           缓存命中?            倒排索引查询
           是 -> 返回           拼音匹配
           否 -> 继续           n-gram匹配
                              前缀匹配
```

### 文档索引流程

```
docs/*.md -> DocParser -> 提取元数据 -> 生成分类索引 -> docs-index.md
                              │
                              v
                        统计信息
                        (示例数/API数)
```

## 性能优化

### 1. 索引优化
- 倒排索引: O(1) 查找
- 预构建: 启动时一次性构建
- 单例模式: 避免重复构建

### 2. 搜索优化
- 快速路径: 精确ID匹配优先
- 拼音支持: 无需切换输入法
- 缓存: 重复查询加速

### 3. 内存优化
- LRU 缓存: 限制内存使用
- 懒加载: 按需解析文档

## 扩展性设计

### 1. 添加新组件
```bash
npm run generate-component -- --name my-component --category form
```

自动生成:
- 文档模板
- 注册表条目

### 2. 添加新搜索方式
1. 在 `extractKeywords` 中提取新特征
2. 在 `getCandidates` 中添加新匹配逻辑
3. 在 `calculateScore` 中调整评分

### 3. 添加新分类
1. 在 `ComponentCategory` 类型中添加
2. 在 `CATEGORY_NAMES` 中添加显示名
3. 更新分类分组逻辑
